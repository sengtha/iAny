# Contribute your voice — classroom Khmer speech collection

A teacher-run way to build an **open Khmer speech-to-text (STT) dataset** with
your students. It lives on its **own page — `https://<your-domain>/voice`** —
separate from the iAny app so the two are never confused. Students read short
Khmer sentences aloud; each recording + its exact text becomes one
`(audio, transcript)` pair.
Feed those pairs into the STT fine-tune (`docs/RUNPOD-KHMER-STT.md`) and the
model understands real Cambodian voices — not just one studio speaker.

**The promise we make to every participant (shown on the screen, in Khmer):**

1. 🗂️ Recordings are released as an **open, free dataset** for anyone to use.
2. 🏅 Everyone who contributes is **credited by name** in the dataset (opt-in).
3. 🆓 The **STT model** built from it is released **free for everyone**.

> **Licensing:** release the dataset and model as **CC-BY-SA-4.0** to match the
> DDD-Cambodia corpus the base model uses, crediting DDD-Cambodia + your
> contributors (`out/CREDITS.md`, generated by the export script).

## For students — how it works

1. Open the **Khmer Voice** page: **`https://<your-domain>/voice`** (share this
   link, a QR code, or a bookmark — it's not inside the iAny app).
2. Fill the short form (all optional except consent): a **credit name** if you
   want to be listed, class/group, age, voice, province.
3. Tick the consent box and tap **Start recording**.
4. A sentence appears. Tap **⏺ Record**, read it **clearly at a natural pace**,
   tap **⏹ Stop**, listen back, then **✓ Keep & next** (or **↺ Re-record**).
5. Repeat. Progress is saved on the device, so a class can pass one phone/laptop
   around and each reader picks up where the sentences run out.

Good clips: quiet room, phone about a hand's width away, no rushing, one sentence
per recording. The app records **16 kHz mono** and trims silence automatically.

## For teachers — setup & privacy

**No extra setup** — the `/voice` page is part of the same deploy. Recordings
upload to your Cloudflare Worker (`/api/voice/clip` → R2 + D1). Make sure the DB
has the table:

```bash
npx wrangler d1 execute iany-radio --remote --file worker/schema.sql
npx wrangler deploy
```

**Privacy (students are minors — treat this as step zero):**

- **Consent is required** before any upload; the screen states the dataset +
  model are released openly. Get **parental/guardian permission** for under-18s.
- The only identifier stored is a random **anonymous voice id** (e.g.
  `s-3f9a2c71`) — never a name. A real name is stored **only** if the student
  opts in to the credit field.
- No emails, phone numbers, or logins are collected.
- You can delete any clip: `DELETE /api/voice/admin/clip/<id>` with the admin token.

**Sentences:** the starter set (~150 sentences covering greetings, school,
numbers, places, health, nature, food, family, questions, and full Khmer script
coverage) lives in `src/assets/voicePrompts.ts`. Add your textbook sentences
there — keep each 3–14 words, pure Khmer script, one clause.

## Exporting the dataset

When you've collected enough, pull everything into a training-ready folder. The
token is the Worker's `RADIO_ADMIN_TOKEN` secret:

```bash
VOICE_ADMIN_TOKEN=your-admin-token \
  node scripts/export-voice.mjs --base https://iany.app --out ./out
```

Produces a **Hugging Face `audiofolder` dataset**, ready to publish:

```
out/clips/<speaker>/<id>.wav   # 16 kHz mono recordings, grouped by voice
out/metadata.csv               # file_name,sentence,speaker,sentence_id,gender,age_band,region,duration_ms
out/README.md                  # dataset card (license, stats, usage)
out/CREDITS.md                 # opt-in contributor names for the release
```

`metadata.csv` matches **§2 of `docs/RUNPOD-KHMER-STT.md`**, so you can fold it
straight into the fine-tune (append your rows to the DDD `meta` list, or train on
it alone for a classroom-voices model). More speakers + real rooms = lower CER on
real audio.

## Publishing the dataset to Hugging Face

The export folder *is* a valid HF dataset — the `file_name` column + dataset card
make it load with `load_dataset(...)`. Publish it (fulfilling the "released as an
open dataset" promise) with the Hugging Face CLI:

```bash
pip install -U huggingface_hub          # once
huggingface-cli login                   # once (a write token from hf.co/settings/tokens)
huggingface-cli upload sengtha/iany-khmer-voice ./out --repo-type dataset
```

The repo is created on first upload (add `--private` to stage it first, then flip
to public in the repo settings). Keep the release **CC-BY-SA-4.0** and the
contributor credits (`README.md` + `CREDITS.md`). Anyone can then use it with:

```python
from datasets import load_dataset
ds = load_dataset("sengtha/iany-khmer-voice")
```

> For a much larger dataset later, convert to Parquet shards for a faster viewer
> and streaming: load the audiofolder and `ds.push_to_hub("sengtha/iany-khmer-voice")`.

## How much do you need?

- A quick classroom model: **a few hours** (e.g. 30 students × 50 sentences).
- Better real-world accuracy: **combine your clips with DDD** (the base) so the
  model gets both broad coverage and your local voices/accents.

Then retrain, convert (GGML for mobile / ct2 for server / ONNX for the PWA), and
publish the model open-source — closing the loop from **your students' voices**
to a **free Khmer STT everyone can use**.
