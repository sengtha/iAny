# Contribute Khmer text photos — community Khmer OCR data

A crowd-sourced way to build an **open Khmer OCR dataset**. It lives on its own
page — **`https://<your-domain>/scan`** — separate from the iAny app. People
photograph Khmer text, the on-device OCR pre-fills what it reads, they **correct
it**, and each `(image, verified text)` pair joins an open dataset for training a
better Khmer OCR — one that reads **real photos**, not just clean rendered fonts.

**The promise to every contributor (shown on the screen, in Khmer):**

1. 🗂️ Photos + text are released as an **open, free dataset** for anyone.
2. 🏅 Contributors are **credited by name** in the dataset (opt-in).
3. 🆓 The **OCR model** built from it is released **free for everyone**.

> **Licensing / privacy:** contributors agree only to photograph text they have
> the right to share — **public signs, printed books, screens** — not private,
> personal, or copyrighted documents. Release the dataset + model **CC-BY-SA-4.0**
> and credit contributors (`out/CREDITS.md`, generated by the export script).

## Why correct-the-machine works
The screen runs the current on-device Khmer OCR and pre-fills its guess, so the
contributor only **fixes the errors** (much less typing than transcribing from
scratch). As a bonus, storing both the guess and the correction shows **exactly
where the current model fails** — the highest-value samples to train on.

## For contributors — how it works
1. Open **`/scan`**, agree to the terms (open dataset + credit).
2. **📷 Take or choose a photo** of Khmer text (a sign, book page, screen).
3. The app reads it and fills in the text — **fix any mistakes**.
4. **Submit & next.** The image is downscaled on-device before upload.

Best photos: clear, straight-on, close-cropped, no glare/blur, page flat.

## For teachers / organizers — setup & privacy
No extra setup — the `/scan` page ships with the same deploy. Samples upload to
the Worker (`/api/ocr/sample` → R2 + D1). Ensure the DB has the table:

```bash
npx wrangler d1 execute iany-radio --remote --file worker/schema.sql
npx wrangler deploy
```

- **Consent required** before any upload; only public/shareable text.
- The only identifier stored is a random **anonymous device id** (e.g.
  `d-3f9a2c71`) — never a name; a real name only if a contributor opts into the
  credit field.
- Delete any sample: `DELETE /api/ocr/admin/sample/<id>` with the admin token.

## Exporting the dataset
The token is the Worker's `RADIO_ADMIN_TOKEN` secret:

```bash
OCR_ADMIN_TOKEN=your-admin-token \
  node scripts/export-ocr.mjs --base https://iany.app --out ./ocr-out
```

Produces:

```
ocr-out/images/<id>.jpg    the photos
ocr-out/metadata.csv       image,text
ocr-out/labels.jsonl       {image, text, ocr_guess, region, w, h} per line
ocr-out/CREDITS.md         opt-in contributor names
```

## Training a better Khmer OCR (later)
With enough real-photo pairs you can **fine-tune the recognizer** (and detector)
on real-world images instead of only synthetic fonts — the same playbook as the
STT: gather diverse, correctly-labeled data, then train and release
**CC-BY-SA-4.0**. `labels.jsonl` (with `ocr_guess`) lets you prioritize the
samples the current model gets wrong.

The loop: **real photos → open dataset → free Khmer OCR** everyone can use.
