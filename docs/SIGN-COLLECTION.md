# Contribute Khmer Sign Language — community KSL data

A crowd-sourced way to build an **open Khmer Sign Language (KSL) dataset**. It
lives on its own page — **`https://<your-domain>/sign`** — separate from the
iAny app. A contributor is shown a Khmer word or letter and **signs it to the
camera**; the on-device hand tracker records the gesture as a short sequence of
hand skeletons, and each `(label, gesture)` pair joins an open dataset for
training a free KSL recognition model.

**The promise to every contributor (shown on the screen, in Khmer):**

1. 🖐️ **Only hand landmarks are recorded — never the video.** No face, no
   background is ever saved, so a contributor can't be identified from a sample.
2. 🗂️ The gestures are released as an **open, free dataset** for anyone.
3. 🏅 Contributors are **credited by name** in the dataset (opt-in).
4. 🆓 The **sign-language model** built from it is released **free for everyone**.

> **Built with and for the Deaf community.** KSL is a real, living language. The
> starter word list (`src/assets/signPrompts.ts`) is only a scaffold — the
> *authoritative* sign for each label comes from Deaf signers, not from us.
> Partner with a Deaf school or association (e.g. the Cambodian Deaf community /
> Krousar Thmey) so the signs collected are correct and the effort is
> community-led. Release the dataset + model **CC-BY-SA-4.0** and credit
> contributors (`out/CREDITS.md`, generated by the export script).

## Why landmarks, not video
The tracker (Google **MediaPipe Hand Landmarker**, Apache-2.0) runs 100%
on-device and turns each frame into **21 (x, y, z) points per hand**. We upload
only those points — a tiny JSON sequence — so:

- **Privacy by design:** there is no image to leak; a hand skeleton is not
  biometrically identifying the way a face is.
- **Tiny + training-ready:** a 2.5 s gesture is a few KB, and landmark sequences
  are exactly what a gesture classifier trains on.

## For contributors — how it works
1. Open **`/sign`**, agree to the terms (landmarks only, open dataset + credit).
2. Allow camera access. Show your hand(s); you'll see a skeleton overlay.
3. You're shown a Khmer word/letter. Press **● Record sign**, wait for the
   3-2-1 countdown, then sign it for ~2.5 seconds.
4. **Submit & next** (or **Redo** / **Skip**). Only the landmarks upload.

Best captures: even lighting, plain background, hand fully inside the frame,
sign at a natural speed.

## For teachers / organizers — setup, assets & privacy
Samples upload to the Worker (`/api/sign/sample` → R2 + D1). Ensure the DB has
the table and deploy:

```bash
npx wrangler d1 execute iany-radio --remote --file worker/schema.sql
npx wrangler deploy
```

**Model asset (one-time):** the hand tracker needs Google's
`hand_landmarker.task` (bundled full model, ~7.5 MB). iAny serves it through the
**same `/models/` mirror as every other model** — the Worker fetches it from
Hugging Face once and caches it in R2, so devices get it same-origin and
offline. Upload the file to a HF repo named **`sengtha/mediapipe-hand`** (the
path already allow-listed in `worker/index.ts`):

```bash
# Download once from the MediaPipe HandLandmarker model card, then:
huggingface-cli upload sengtha/mediapipe-hand hand_landmarker.task
```

The app requests it at
`/models/sengtha/mediapipe-hand/resolve/main/hand_landmarker.task`. For local
testing you can point at any URL via `localStorage.setItem('iany.handModel', …)`.

The MediaPipe **WASM** runtime is copied automatically into `public/mediapipe/`
by `scripts/copy-ort.mjs` (runs before dev/build), so it's served same-origin too.

- **Consent required** before any upload.
- The only identifier stored is a random **anonymous device id** (e.g.
  `g-3f9a2c71`) — never a name; a real name only if a contributor opts into the
  credit field.
- Delete any sample: `DELETE /api/sign/admin/sample/<id>` with the admin token.

## Exporting the dataset
The token is the Worker's `RADIO_ADMIN_TOKEN` secret:

```bash
SIGN_ADMIN_TOKEN=your-admin-token \
  node scripts/export-sign.mjs --base https://iany.app --out ./sign-out
```

Produces:

```
sign-out/sequences/<id>.json   per-frame hand-landmark sequences (no video)
sign-out/labels.csv            seq,label
sign-out/labels.jsonl          {seq, label, prompt_id, frames, hand_frames, region} per line
sign-out/label-counts.csv      how many samples per sign (spot thin coverage)
sign-out/CREDITS.md            opt-in contributor names
```

## Training a KSL recognizer (later)
Each sample is a variable-length sequence of hand landmarks with a label — the
classic setup for gesture recognition:

- **Normalize** each frame (translate to the wrist, scale by hand size) so the
  model is position/scale invariant.
- **Static signs** (letters, digits): a small MLP on a representative frame, or
  averaged landmarks, works well.
- **Dynamic signs** (words with motion): feed the sequence to a small
  temporal model (1D-CNN / GRU / Transformer) over the per-frame features.
- Aim for **many signers per label** (like STT needs many speakers) — check
  `label-counts.csv` and keep collecting where coverage is thin.

Then release the model **CC-BY-SA-4.0**, on-device (runs anywhere MediaPipe
does). The loop: **real signers → open dataset → free Khmer Sign Language
recognition** everyone can use.
